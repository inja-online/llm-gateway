## How Cursor connects

Cursorâ€™s **Settings â†’ Models** can:

1. Take an **OpenAI API Key** (any string you choose as the gateway edge key).  
2. **Override OpenAI Base URL** to your gatewayâ€™s OpenAI dialect root (must include **`/v1`**).  
3. List **custom model names** you add â€” these are sent as the `model` field and must match gateway **aliases** or `provider/model` ids.

Cursor does **not** use Claude Codeâ€™s `ANTHROPIC_BASE_URL` env. It only talks **OpenAI-shaped** HTTP to the override URL. The gateway still routes Claude/Grok/GPT by **model id**.

## Prerequisites

1. Built binary: `go build -o llm-gateway ./cmd/gateway`  
2. At least one subscription logged in:

```bash

./llm-gateway auth login claude     # optional
./llm-gateway auth import grok      # or auth login grok
./llm-gateway auth status
```

## Cursor Settings (GUI)

Open **Cursor Settings â†’ Models**.

### 1. OpenAI API Key

Enable **OpenAI API Key** and paste:

```text
local-dev
```

(Use the same value as `KEY` / edge key. Not your OpenAI Platform `sk-` â€” the gateway holds subscription OAuth.)

### 2. Override OpenAI Base URL

Enable **Override OpenAI Base URL** and paste:

```text
https://127.0.0.1:8787/v1
```

| Correct | Wrong |
|---------|--------|
| `https://127.0.0.1:8787/v1` | `https://127.0.0.1:8787` (missing `/v1`) |
| `https://localhost:8787/v1` | `http://â€¦` if the gateway is TLS-only |

Cursor calls `{base}/chat/completions` and, in Agent mode, often `{base}/responses`.

### 3. Add models

**Add Model** for each name you want in the picker. Names must match the gateway config aliases:

| Model name in Cursor | Upstream (Jul 2026) |
|----------------------|---------------------|
| `gpt` / `terra` | ChatGPT `gpt-5.6-terra` |
| `sol` | ChatGPT `gpt-5.6-sol` |
| `luna` | ChatGPT `gpt-5.6-luna` |
| `grok-4.5` | xAI `grok-4.5` |
| `composer-2.5` | xAI `grok-build-0.1` (Composer-class / Grok Build) |
| `sonnet` / `opus` / `haiku` | Claude Sonnet 5 / Opus 4.8 / Haiku 4.5 |

Full ids: `chatgpt/gpt-5.6-terra`, `xai/grok-4.5`, `xai/grok-build-0.1`, `anthropic/claude-sonnet-5`, â€¦

Click **Verify** if available, then select the model in Chat or Agent.

### Combos

Only add models for subscriptions you authenticated:

| Goal | Models to enable in Cursor |
|------|----------------------------|
| GPT only | `gpt` / `terra` / `sol` / `luna` |
| Grok only | `grok-4.5` / `composer-2.5` |
| GPT + Grok | both families |
| + Claude | also `sonnet` / `opus` / `haiku` / `fable` |

You do **not** need Cursorâ€™s separate Anthropic API key field for Claude-through-gateway. Send Claude as a **custom OpenAI model name** (`sonnet`); the gateway routes OpenAI dialect â†’ Anthropic (passthrough or translate as configured).

## TLS / certificates

`cc-gateway-up` serves **HTTPS** with certs from `examples/scripts/gen-localhost-tls.sh`.

```bash
# Best trust story for Electron/Cursor:
brew install mkcert
mkcert -install
./examples/scripts/gen-localhost-tls.sh
cc-gateway-down && cc-gateway-up
```

If **Verify** fails with a certificate error, fix local trust (mkcert) rather than disabling TLS.

## What works (and limits)

| Surface | Expected |
|---------|----------|
| Cursor Chat / Plan / Ask | OpenAI Chat Completions â†’ gateway |
| Cursor Agent | Often **Responses API** (`POST /v1/responses`) â€” supported for `openai` / `openai_compat` (ChatGPT, xAI) |
| Claude models via Cursor | OpenAI-shaped request with Claude model id â†’ gateway routes to Anthropic |
| Image / multimodal BYOK | Some Cursor versions ignore custom base URL for images â€” text/agent is the supported path |
| Writing keys via script | **Not supported** by Cursor (secure storage / GUI only) |

Same gateway process can serve **Claude Code** and **Cursor** at once (`ANTHROPIC_BASE_URL` for Claude Code, OpenAI override for Cursor).

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| Connection refused | `cc-gateway-up`; check `curl -sk https://127.0.0.1:8787/healthz` |
| 404 on models | Base URL missing `/v1`, or model name not in gateway aliases |
| 401 | `KEY` / OpenAI API Key mismatch with edge auth; or subscription not logged in |
| Verify TLS error | `mkcert -install` + regenerate certs |
| Agent fails, Chat works | Agent may need `/v1/responses`; ensure ChatGPT/xAI providers are `openai` / `openai_compat` |
| Wrong model family | Check alias targets in `examples/configs/claude-code-subscriptions.yaml` |

```bash
cursor-verify
./llm-gateway auth status
curl -sk https://127.0.0.1:8787/v1/models -H "Authorization: Bearer local-dev"
```

## Architecture

1. Cursor sends OpenAI dialect to `https://127.0.0.1:8787/v1`.  
2. Gateway resolves `model` â†’ alias â†’ `provider/model`.  
3. Auth uses subscription store (`oauth.credentials: chatgpt|claude|grok`).  
4. Claude Code can use the **same** process with Anthropic dialect on the same host.
